✅ Core Design Principles – Implemented

  • Shopify as system of record for customers and billing
  • Custom backend for circulation, timers, swaps, photos
  • No SKUs, no cart line items, no stock separation
  • One national inventory pool with universal access
  • Speed, simplicity, staff ergonomics prioritized

✅ Subscription Tiers & Rules – Implemented

  • Basic: 1 item/month, 0 swaps, max 1 item out
  • Plus: 5 items/month, 2 swaps/month, max 2 items out
  • Premium: 10 items/month, 5 swaps/month, max 4 items out
  • Monthly counters reset on billing cycle

✅ Data Model – Implemented

  • Members table: card token, Shopify customer ID, tier, status, cycle dates, counters
  • Loans table: member ID, store location, checkout time, due date, return time, photo URL
  • Audit events table: records all actions for traceability

✅ In-Store Flows (Backend APIs) – Implemented

  • ✅ POS retrieves member status, allowances, items out – Backend ready
  • ✅ Backend creates loan record linking photo to member – Backend ready
  • ✅ Returns flow – Backend ready
  • ✅ Swap flow – Backend ready

✅ Safeguards – Implemented

  • Block checkout when subscriptions inactive
  • Block when allowances exceeded
  • Block when max items out reached
  • All actions logged for auditing

✅ Subscription Lifecycle Sync – Implemented

  • Backend listens to Shopify subscription webhooks
  • Keeps member status, tier, billing cycles in sync
  • Monthly counters reset automatically

⚠️ Returns Flow – Backend Ready, UI Missing

  • ❌ Staff scans card, visually matches returned item to photo thumbnail, confirms return – No POS extension yet
  • ✅ Backend closes loan and frees capacity – POST /api/loans/return

⚠️ Swap Flow – Backend Ready, UI Missing
